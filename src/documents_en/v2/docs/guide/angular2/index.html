For easier installation, Monaca CLI will download all development & build dependencies including TypeScript compiler, Webpack and polyfills. ##### Use Monaca CLI ```bash $ npm install -g monaca # Install Monaca CLI - Onsen UI toolkit $ monaca create helloworld # Choose Angular 2 template $ cd helloworld; monaca preview # Run preview, or "monaca debug" to run on your device ``` ##### Download via npm ```bash # You need to install Onsen UI and Onsen UI for Angular 2 $ npm install onsenui angular2-onsenui ``` ### Loading Onsen UI in Angular 2 You need to import `onsenui` and `angular2-onsenui` package as shown below. ```javascript // Angular 2 imports import '@angular/platform-browser-dynamic'; import '@angular/platform-browser'; import '@angular/core'; import '@angular/http'; import '@angular/router'; // Onsen UI window['ons'] = require('onsenui/js/onsenui'); import { ONS_DIRECTIVES, OnsNavigator } from 'angular2-onsenui'; ``` ### Onsen UI Directives and Web Components Onsen UI for Angular 2 components are Angular 2 Directives. You need to specify the directives to use at `directives` in `@Component` decorator like below: ``` import { OnsInput, OnsPage } from `angular2-onsenui` // Example Component with Onsen UI for Angular 2 @Component({ selector: 'app', directives: [ OnsInput, OnsPage ], template: `Click Me!` }) export class AppComponent { // Application code should be written in here } ``` In this case, we are using `OnsInput` and `OnsPage` directive, which represents `ons-input` and `ons-page` tag. You might noticed that `ons-button` is not specified in the directive. This is because this component is not a Angular 2 directive, but is a part of our JavaScript Core implemented as Web Components. When a component is defined with a modifier, it will have a separate class namespace so that you can apply custom styles to the component. Also, some components have several preset modifiers to change the appearance. All preset modifiers are described in the components' reference. ``` Quiet Light Large CTA ``` ### Using components Both Directives and Web Components have methods, which can be called from the program after the components are initialized. Directives will be still undefined in the caller's constructor, so one option is to use `ngAfterViewInit()` method. ``` export class AppComponent implements AfterViewInit { ngAfterViewInit() { this._navigator.pushComponent(component, { animation: 'fade' }); } ``` For Web Components, the methods are directly attached to the DOM element. For instance, you can use the traditional `querySelector()` to query the element and call the method. For Angular 2, you can also use template reference variable (using `#` character) to refer to a DOM element. For details, please read (Angular 2 Guide)[https://angular.io/docs/ts/latest/guide/template-syntax.html#!#ref-vars]. ### Components with child view Some components like `OnsNavigator` and `OnsSplitter` have child view.
