For instance, [``](/reference/ons-page.html) provides a set of DOM events that will be fired in different moments of its life cycle. Use these events to alter the behavior on each page. * `init` event is fired after `` is attached to DOM. * `destroy` event is fired before `` is destroyed and prior to DOM detachment. * `show` event is fired every time `` comes into view, i.e. when a new page is created and shown immediately or when an existing page shows up. * `hide` event is fired every time `` disappear from view, i.e. when a visible page is destroyed or is hidden but still exists in the page stack. Page lifecycle events will be propagated to the page's descendants so they are correspondingly shown, hidden or destroyed.
